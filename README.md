# ПОЯСНИТЕЛЬНАЯ ЗАПИСКА

## К РЕЗУЛЬТАТАМ ВЫПОЛНЕНИЯ ПРАКТИЧЕСКОГО ЗАДАНИЯ ДЕМОНСТРАЦИОННОГО ЭКЗАМЕНА

### Разработка информационного модуля для магазина «Velo Shop / Aba»

---

## 1. Общее описание системы

Разработанное программное обеспечение представляет собой десктопное приложение для автоматизации торговой деятельности магазина. Система обеспечивает разграничение прав доступа, ведение товарного каталога, управление запасами, а также оформление и обработку заказов клиентов.

Взаимодействие с базой данных реализовано через объектно-ориентированный модуль, исключающий прямую хардкод-зависимость, а интерфейс построен на современных принципах динамического формирования контента (карточек товаров).

### Технологический стек проекта:

* **Язык программирования:** Python 3.x
* **Графический интерфейс:** PyQt6 (компоненты `QMainWindow`, `QTableWidget`, `QComboBox`, `QFrame`, `Layouts`)
* **Стилизация интерфейса:** Библиотека `qt-material` (тема `light_purple.xml`), кастомные таблицы стилей QSS для интерактивных элементов.
* **СУБД:** Реляционная база данных (используются параметризованные SQL-запросы для защиты от SQL-инъекций).

---

## 2. Архитектура и структура программного кода

Приложение спроектировано с использованием принципов ООП и механизма наследования:

* **`BaseWindow(QMainWindow)`** — базовый класс, содержащий универсальные методы, которые используются на разных экранах приложения (очистка динамических виджетов, заполнение таблиц, фильтрация данных, загрузка категорий и поставщиков из БД).
* **`LoginWindow`** — стартовое окно аутентификации пользователей.
* **`AdminWindow`** — рабочее место администратора/менеджера (наследует `BaseWindow`).
* **`ClientWindow`** — интерфейс для покупателей (наследует `BaseWindow`).
* **`GuestWindow`** — гостевой режим с ограниченными правами (наследует `BaseWindow`).

---

## 3. Функциональные возможности по ролям

Система гибко настраивает интерфейс (скрывает/отображает вкладки и кнопки) в зависимости от уровня доступа вошедшего пользователя.

### А. Роль «Администратор / Менеджер»

Обеспечивает полный контроль над бизнес-процессами магазина:

* **Мониторинг заказов:** Вывод полной информации о заказах (ID, ФИО клиента, товар, размер, количество, общая сумма, статус, дата) в интерактивную таблицу.
* **Управление статусами:** Смена статуса заказа («обработка», «отправлено», «доставлено», «возврат») через выпадающий список с сохранением изменений в БД.
* **Управление каталогом:** Поиск товаров по названию (динамический по мере ввода текста), фильтрация по категориям и по поставщикам одновременно.
* **Удаление данных:** Полное каскадное удаление товаров (сначала удаляются зависимые спецификации/варианты, затем сам товар) и удаление заказов. Реализована защита транзакций (`commit/rollback`) и предупреждающие диалоговые окна `QMessageBox`.

### Б. Роль «Клиент»

Ориентирована на удобный выбор и безопасную покупку:

* **Интерактивный каталог:** Просмотр доступного ассортимента.
* **Выбор модификаций:** При выборе товара динамически подгружаются его варианты (доступные размеры, цвета и их остаток на складе).
* **Автоматический расчет стоимости:** Алгоритм на лету калькулирует итоговую цену по формуле:

$$\text{Итого} = ((\text{Базовая цена} + \text{Модификатор варианта}) \times (1 - \frac{\text{Скидка}}{100})) \times \text{Количество} + \text{Стоимость доставки}$$


* **Оформление заказа:** Проверка корректности ввода адреса и валидация остатков на складе. Если товара недостаточно, система блокирует покупку и выдает предупреждение. При успешном заказе остаток на складе автоматически уменьшается.

### В. Роль «Гость»

* Демонстрационный режим, позволяющий исключительно просматривать базовый каталог товаров без возможности фильтрации, поиска и покупки.

---

## 4. Особенности интерфейса и бизнес-логики

В коде реализованы важные UI/UX решения для повышения эргономики:

* **Визуальное кодирование (Подсветка карточек):**
* Если товара **нет в наличии** (общий остаток всех вариантов равен 0), карточка окрашивается в **голубой цвет** (`#87CEEB`), сигнализируя о необходимости поставки.
* Если на товар установлена **акционная скидка более 15%**, карточка окрашивается в **зеленый цвет** (`#2E8B57`) для привлечения внимания покупателя.


* **Отображение цен:** Если у товара есть скидка, старая цена красиво зачеркивается красным цветом, а новая выводится жирным шрифтом.

---

## 5. Инструкция по авторизации (для экспертов / проверки)

Для проверки работоспособности ролевой модели в окне авторизации используются учетные записи, хранящиеся в таблице `users` базы данных.

> **Важное техническое замечание:** В текущей версии для демоэкзамена сверка паролей в SQL-запросе происходит напрямую (по полю `u.password_hash = %s` с передачей открытого текста из `QLineEdit`). Пароли вводить в исходном виде, как они записаны в БД.

### Тестовые данные для входа:

| Логин (username) | Пароль | Роль в системе | Доступный функционал |

| **client_demo** *(или ваш логин)* | *ваш пароль из БД* | Клиент | Выбор товара, калькуляция цены, проверка склада, оформление заказа. |
| Кнопка **«Войти как гость»** | Не требуется | Гость | Только просмотр базового списка товаров. |
