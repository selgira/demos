Пояснительная записка к практическому заданию демонстрационного экзамена по разработке информационного модуля для магазина Вело Шоп Аба. Разработанное программное обеспечение представляет собой десктопное приложение для автоматизации работы магазина, написанное на языке программирования Python с использованием библиотеки PyQt6 для создания графического интерфейса. Стилизация приложения выполнена с помощью библиотеки qt-material с темой light_purple, а подключение к базе данных реализовано через отдельный объектно-ориентированный модуль Database, что обеспечивает безопасность выполнения запросов и защиту от сбоев.

Архитектура приложения построена на принципах объектно-ориентированного программирования и механизме наследования классов. Главным в системе является базовый класс BaseWindow, в котором прописаны универсальные методы для очистки списков, заполнения таблиц данными, а также загрузки категорий товаров и поставщиков из базы данных в выпадающие списки. Остальные окна приложения, такие как окно администратора AdminWindow, окно клиента ClientWindow и окно гостя GuestWindow, наследуются от этого базового класса, перенимая его функции. Стартовым окном для запуска программы является окно авторизации LoginWindow, которое отвечает за проверку учетных данных.

Интерфейс программы динамически настраивается в зависимости от роли пользователя, вошедшего в систему, а все лишние элементы скрываются кодом. Администратор или менеджер имеет самые полные права в системе. Он может просматривать интерактивную таблицу со всеми заказами, где отображаются данные клиента, выбранный товар, размер, количество, итоговая сумма, статус и дата. Также администратор может менять статус заказа через выпадающий список, искать товары по названию прямо во время ввода текста, фильтровать их одновременно по категориям и поставщикам, а также удалять заказы и товары. При этом в коде настроено безопасное каскадное удаление вариантов товара перед удалением самой главной записи, чтобы не возникало ошибок связи в базе данных, а перед самим удалением всегда всплывает окно подтверждения.

Роль клиента предназначена для совершения покупок, поэтому в его окне отображается каталог товаров, где после выбора конкретной позиции динамически подгружаются доступные варианты размеров, цветов и их остаток на складе. Калькулятор в реальном времени рассчитывает итоговую стоимость заказа, складывая базовую цену товара и наценку за размер, вычитая процент скидки, умножая на количество и добавляя стоимость выбранного способа доставки. При оформлении заказа система проверяет заполнение адреса и наличие нужного количества товара на складе, блокируя покупку в случае нехватки и автоматически уменьшая остатки в базе данных при успешном оформлении.

Гостевой режим является самым простым и предоставляет пользователю исключительно возможность просмотра каталога товаров, при этом все функции поиска, фильтрации по поставщикам и оформления заказов полностью скрываются из интерфейса, чтобы гость не имел к ним доступа.

В каталоге также реализовано визуальное кодирование для удобства восприятия информации. Благодаря этому карточки товаров, которых вообще нет на складе, автоматически окрашиваются в голубой цвет, сигнализируя о необходимости новой поставки. Товары с хорошей скидкой более пятнадцати процентов подсвечиваются зеленым цветом для привлечения внимания покупателей. Если на товар действует акция, то старая цена отображается зачеркнутой и красной, а новая выводится жирным шрифтом.

Для проверки работоспособности программы эксперты могут использовать учетные записи из таблицы пользователей в базе данных. Сверка паролей в коде происходит прямым текстом из полей ввода без предварительного хеширования, поэтому данные нужно вводить точно так же, как они записаны в строках базы. Для входа под администратором или клиентом вводятся соответствующие логины и пароли из таблицы, а для гостевого режима вводить учетные данные не требуется, достаточно просто нажать кнопку входа для гостя.
